03 — Prinsip Desain
Prinsip & Standar
Aturan yang harus dipahami oleh seluruh tim desain dan developer sebelum membuat atau mengubah halaman Sisakti+.
Prinsip Inti
01
Clarity
Jelas Tanpa Noise
Setiap elemen harus punya tujuan. Hindari dekorasi berlebihan. Prioritaskan keterbacaan informasi SKP di atas estetika semata.
02
Trust
Membangun Kepercayaan
Warna hijau forest, tipografi serif yang kokoh, dan palet earth tone menciptakan rasa profesional dan dapat dipercaya untuk sistem validasi akademik.
03
Efficiency
Efisiensi Kerja
Admin BEM perlu memvalidasi banyak sertifikat cepat. Desain harus mengurangi friction: tombol approve/reject prominent, filter mudah diakses.
04
Feedback
Umpan Balik Segera
Setiap aksi harus memberikan respons visual: badge berubah, warna status, notifikasi muncul. Mahasiswa harus tahu statusnya tanpa bertanya.
05
Hierarchy
Hierarki Visual
Total SKP adalah angka terpenting — tampilkan dengan Fraunces 28px bold. Label sekunder dengan DM Sans 11px. Jangan pernah membalik hierarki ini.
06
Role-based
Konteks Berbasis Peran
Mahasiswa, Admin BEM, dan Superadmin memiliki kebutuhan berbeda. Sidebar, warna aksen notifikasi, dan CTA utama harus mencerminkan peran pengguna.
Aturan Layout
Grid System
• Sidebar: 240px fixed, lebar tetap
• Konten utama: flex: 1, mengisi sisa
• Total canvas: 1280px width
• Padding konten: 28px semua sisi
• Grid 2 kolom: gap 16px
• Grid 4 kolom: gap 14px
Komponen Hierarchy
1. Topbar — sticky, selalu terlihat
2. Metric cards — 4 kolom di atas halaman
3. Grid 2 kolom — konten utama
4. Full-width card — tabel dan list
5. Sidebar — navigasi sekunder
Warna Status (Wajib)
• Disetujui: #1D9E75 Green
• Ditolak: #C0392B Red
• Pending: #B8860B Gold
• Wajib/Info: #2563EB Blue
• Notif badge: #E05C2A Orange
Z-Index Stack
• Sidebar: z-index 100
• Topbar: z-index 50
• Modal overlay: z-index 300
• Notif dropdown: z-index 200
• Role switcher (demo): z-index 200
Do & Don't
✓ DO — Warna status konsisten
Disetujui Pending Ditolak
Selalu gunakan warna semantik yang sama untuk status yang sama di seluruh halaman.
✗ DON'T — Warna acak-acakan
Disetujui Pending Ditolak
Jangan gunakan warna berbeda untuk status yang sama. Pengguna akan bingung membaca feedback visual.
✓ DO — Label huruf kapital kecil
Kategori SKP
Progress per Kategori
Gunakan uppercase untuk label form. Gunakan sentence case untuk judul card.
✗ DON'T — Inkonsistensi casing
Kategori Skp
PROGRESS PER KATEGORI
Jangan campur aturan casing secara acak. Judul card tidak perlu all-caps.
✓ DO — Fraunces untuk angka besar
82 / 120
Angka SKP utama selalu pakai Fraunces serif. Angka sekunder / total pakai DM Sans kecil.
✗ DON'T — DM Sans untuk metric utama
82 / 120
Jangan gunakan DM Sans tebal untuk angka metric utama. Kehilangan karakter dan keterbacaan.
✓ DO — Sidebar sidebar always visible
Sidebar 240px selalu tampil di sisi kiri. Navigasi tetap konsisten di semua halaman dalam satu role. Active state selalu terlihat.
Pengguna perlu orientasi yang konsisten saat berpindah halaman.
✗ DON'T — Collapsed sidebar tanpa trigger
Jangan sembunyikan sidebar tanpa indikator yang jelas. Jangan ubah urutan menu antar halaman. Jangan hilangkan active state.
Sidebar yang tidak konsisten meningkatkan cognitive load pengguna.

Aksesibilitas & Standar Kualitas
🎨
Color Contrast
Semua teks body memenuhi WCAG AA (4.5:1). Teks di atas background hijau forest (#1B4D3E) menggunakan putih atau rgba(255,255,255,0.9).
📱
Touch Target
Minimum ukuran tombol dan elemen interaktif adalah 36×36px. Tombol approve/reject di queue item minimal 80px lebar agar mudah diklik.
⌨️
Focus State
Setiap input dan tombol harus punya focus ring yang jelas: box-shadow 0 0 0 3px rgba(74,140,114,0.25). Jangan hilangkan outline bawaan browser tanpa pengganti.
📊
Feedback Visual
Setiap perubahan status harus ada visual feedback: warna badge berubah, toast notification muncul, atau elemen fade-out saat selesai diproses.
🔤
Font Minimum
Ukuran font minimum di seluruh aplikasi adalah 11px untuk label uppercase. Konten body minimal 13px. Jangan gunakan font di bawah 11px untuk informasi apapun.
⚡
Loading States
Saat data sedang dimuat, tampilkan skeleton placeholder atau spinner. Jangan biarkan konten "loncat" setelah data loaded. Gunakan consistent height untuk placeholder.
Empty States
📭
Belum ada sertifikat
Upload sertifikat pertama kamu untuk mulai mengumpulkan SKP.
✅
Antrian kosong
Semua sertifikat sudah divalidasi. Tidak ada yang menunggu antrian.
🔍
Tidak ditemukan
Tidak ada hasil untuk pencarian "nama mahasiswa".
